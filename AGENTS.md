# Agent Instructions

This repository may become public. Keep all changes suitable for an open-source project.

Before making changes, read:

- [README.md](README.md)
- [docs/product-thesis.md](docs/product-thesis.md)

## Open-Source Posture

- Do not add proprietary client details.
- Do not name confidential companies, systems, repositories, or engagements.
- Keep examples generic or clearly fictional.
- Prefer reusable patterns over consultancy-specific internal language.
- Capture decisions in public-safe documents under `docs/`.

## Repository Conventions

- Keep the README concise and product-facing.
- Put rationale, strategy, and tradeoffs in `docs/`.
- Prefer small, inspectable files over broad framework decisions at this stage.
- Do not introduce implementation dependencies until the project structure is clearer.
- When adding examples, make them generic and self-contained.
