Architecture harness for AI coding agents

Expert architecture knowledge, built into every AI coding agent.

Architect Companion packages the practices that experienced software architects rely on as reusable profiles. Pick the profiles that fit your project, compose them into one harness, and render that knowledge straight into the configuration your AI coding tools read. The standards reach every tool automatically, instead of being pasted in by hand.

Profiles

Architect Companion ships with profiles, and each one captures architectural knowledge from embarc's software architecture practice: the principles, policies, workflows, review heuristics, and worked examples for a particular kind of system. This is the knowledge experienced architects bring to a project, written down once and kept reusable.

Pick the profiles that match what you are building, then compose them with your own project. You get a harness shaped to your setup, not a blank configuration file you have to fill in yourself.

cli
Layered command line applications with a thin adapter at the edge.
typescript
Typed boundary contracts and stable public entrypoints.
modular-monolith
Clear module boundaries inside a single deployable.

Why

AI coding tools each read their own configuration. CLAUDE.md, AGENTS.md, Cursor rules, and the rest fall out of sync the moment someone edits one by hand. The boundaries you agreed on in one file quietly contradict the next.

Architect Companion keeps a single source of architectural truth and renders the rest from it, so the same boundaries and policies reach every tool. It does not run reviews or write code. It configures the tools that do.

What it renders

AGENTS.md
Instructions for AI agents working in the repository.
CLAUDE.md
Project instructions for Claude Code.
.cursor/rules/architect-companion.mdc
Rules read by Cursor.
.dependency-cruiser.cjs
Dependency boundary checks you can run locally and in CI.
.github/workflows/architecture.yml
An architecture check that runs on pull requests.
.pr_agent.toml
Review configuration for PR-Agent.
.claude/settings.json
Claude Code hooks that stop generated files from being hand-edited.

How it works

One harness file names the profiles that supply the knowledge, your project languages, the architectural boundaries, and which targets to render. Everything else is generated from it.

architect-companion render ↓
  • AGENTS.md
  • CLAUDE.md
  • .cursor/rules/architect-companion.mdc
  • .dependency-cruiser.cjs
  • .github/workflows/architecture.yml
  • .pr_agent.toml
  • .claude/settings.json

The tool exposes a handful of commands.

init
Scaffold the harness and render the enabled targets.
render
Generate every target file from the resolved model.
render --check
Re-render and fail if any committed file has drifted.
doctor
Diagnose missing tools, warnings, and profile lock status.

Verify in CI

render --check re-renders every target and compares it to what is committed. Wire it into your pipeline and a pull request cannot merge with stale configuration. 

$ architect-companion render --check
Generated targets are fresh.

Get started

The project is not on npm yet. Clone it, build it, and run the CLI from the repo.

$ git clone https://github.com/Architect-Companion/architect-companion
$ cd architect-companion
$ npm ci
$ npm run build
$ node dist/cli.js --help

Then run init inside a project to scaffold the harness, and render whenever the model changes. The documentation covers the harness model, profiles, and the boundary between deterministic checks and AI-assisted review.
